# Step 3 光源与目标色确认优化设计

日期：2026-07-11

## 结论

采用方案 B：在不接入真实测色仪、不生成生产配方的前提下，增强「3. 确认目标色与打样参数」的业务准确性和可视化判断能力。

本次设计把 Step 3 从单一 Lab 色块表单升级为「AI 初始提取参数 + 人工确认实时值 + 历史库参考值 + 多光源实时渲染 + 图片提示词 + 清晰确认参数」的一屏确认工具。

## 背景

当前项目是 ColorBridge MVP，主入口为 Next.js 工作台。Step 3 当前能力包括：

- 一个目标 Lab 色块。
- Lab 的 L/a/b 三轴 slider。
- 更深、更浅、少红、更蓝、少蓝快捷微调。
- 光源选项：D65、D50、TL84、A 光源。
- 基布、Delta E 阈值、Lab 数值输入。

问题是：当前界面能编辑 Lab，但不能充分支持纺织打样里的判断场景。用户需要看到不同光源、角度、强度下的趋势，并清楚区分 AI 初始提取值、人工确认后的实时值和历史库参考值。

## 调研依据

本设计参考以下色彩与光谱方向资料：

- 同色异谱说明：材料表面颜色由材料反射光谱和光源光谱共同决定；纺织行业常见光源同色异谱、观察者同色异谱和视场同色异谱，多个光源比较是必要判断手段。  
  https://en.wikipedia.org/wiki/Metamerism_(color)
- CIE 标准光源说明：除 D65、D50、A、F 系列外，还有 LED 与室内日光 ID 系列。MVP 不需要完整光谱库，但应扩展常用场景光源。  
  https://en.wikipedia.org/wiki/Standard_illuminant
- 色差呈现说明：色差公式和视觉判断会受呈现条件影响，目标色与样品相邻展示比单独色块更利于人工判断。  
  https://arxiv.org/abs/1904.11293
- Lab/sRGB 到反射光谱存在多解，不能从 Lab 反推出唯一真实光谱。因此本项目只能做趋势模拟和风险提示，不能声称为实测光谱结果。  
  https://arxiv.org/abs/1710.05732
- 光源 SPD 可显著改变材料外观，未来可扩展到真实光谱控制或测量，但不纳入当前 MVP。  
  https://arxiv.org/abs/2407.09511

## 方案对比

### A. 只优化 UI 展示

核心思路：只在现有字段上扩展视觉呈现，不改确认数据结构。

改动范围：Step 3 页面局部。

优势：最快，风险最低。

风险：主光源、复核光源、观察角度、图片提示词等业务信息无法稳定沉淀，只能显示，不能追踪。

工作量：小。

异常路径：历史库未命中时继续展示 AI 初始 Lab。

### B. UI + 确认 JSON 扩展

核心思路：保留现有 `confirmedFields` JSON 存储方式，扩展确认结构，保存光源、观察条件、图片提示词、历史参考 fallback 来源。

改动范围：Step 3 UI、确认草稿类型、确认校验、确认结果 JSON、测试。

优势：既能让界面直观，也能保留业务决策依据；不需要新增数据库表或测色仪接入。

风险：需要谨慎处理旧确认数据兼容，避免已有 demo 状态无法读取。

工作量：中。

异常路径：历史库无参考值时，明确使用 AI 初始提取值作为 fallback，并在 UI 标注「待历史样验证」。

### C. 预留真实光谱/测色仪接口

核心思路：新增光谱数据模型和设备接口预留。

改动范围：数据模型、接口、UI、测试都要扩展。

优势：长期更完整。

风险：超出当前 Demo 目标；没有真实设备和光谱数据时容易变成伪精确。

工作量：大。

异常路径：没有设备数据时回退到模拟渲染。

## 选择

选择 B。

原因：用户要求业务准确性优先，同时要求实时、直观。方案 B 能保存确认依据，也不会引入当前项目无法验证的真实光谱能力。

## 用户界面设计

Step 3 保持一屏工作台，不做新页面。

左侧为视觉判断区：

- 三张色卡：
  - AI 初始提取参数：显示客户语义、AI 提取 Lab、默认预览光源。
  - 人工确认实时值：显示调整后的 Lab、主光源、色温、光照强度、观察角度、材质影响。
  - 历史库参考值：命中数据库时显示历史样；未命中时显示 AI 初始提取值，并标注 fallback。
- 实时渲染预览：
  - 用当前 Lab、主光源、色温、光照强度、观察角度、纹理/光泽参数生成屏幕预览。
  - 同屏展示 D50、D65、A 光等简化对比色块。
  - 明确标注「趋势模拟，不替代实测光谱」。
- 图片提示词紧凑条：
  - 放在实时渲染区下方。
  - 不做大面板。
  - 显示颜色描述、图片风险和追问建议的摘要。
- 快捷微调：
  - 更深 L-1。
  - 更浅 L+1。
  - 减红 a-1。
  - 加蓝 b-1。
  - 减蓝 b+1。

右侧为参数确认区：

- 光源与观察条件：
  - 主光源：D65、D50、TL84/F11、A、LED-B3、ID65。
  - 复核光源：可选，默认建议 TL84/F11。
  - 模拟日光色温 CCT slider。
  - 光照强度 slider。
  - 观察角度 slider。
  - 纹理/光泽影响 slider。
- 确认参数：
  - 必须使用白底、深色文字、清晰边框。
  - 字段包括主光源、复核光源、基布、目标 Lab、容差、确认备注。
  - 禁止使用暗色背景输入框。
- 风险与下一步：
  - 展示同色异谱风险等级。
  - 历史库未命中时提示「待历史样验证」。
  - 历史库命中后替换参考值和风险提示。

## 数据结构

保留现有 `ConfirmedRequirement` 的核心字段：

- `illuminant`
- `baseCloth`
- `targetLab`
- `deltaEThreshold`

扩展确认 JSON，新增字段：

- `reviewIlluminant`: 复核光源，例如 `TL84/F11`。
- `lighting`: 光源模拟参数。
  - `cctKelvin`
  - `illuminanceLux`
  - `viewingAngle`
  - `textureGloss`
- `toleranceMode`: 容差模式，MVP 可为 `deltaE76` 或 `cmc`.
- `confirmationNote`: 人工确认备注。
- `imagePromptHints`: 图片提示词摘要。
  - `colorDescription`
  - `imageRisk`
  - `materialHint`
  - `followUpSuggestion`
- `referenceSource`: 历史参考来源。
  - `history-case`
  - `ai-extracted-fallback`

兼容策略：

- 读取旧数据时，缺失的新字段使用界面默认值。
- 保存新数据时，仍写入 `ColorOrder.confirmedFields` JSON，不新增数据库表。
- 历史库未命中时，`referenceSource` 设置为 `ai-extracted-fallback`。

## 数据流

1. Step 2 产出 AI 结构化分析。
2. Step 3 初始化草稿：
   - Lab 优先使用已确认值。
   - 其次使用 AI 提取值。
   - 历史参考未命中时，参考色块使用 AI 提取值。
3. 操作员调整 Lab、主光源、复核光源、CCT、lux、角度、纹理/光泽。
4. UI 实时更新人工确认色块和渲染预览。
5. 操作员确认参数。
6. Server action 保存扩展后的 `confirmedFields` JSON。
7. Step 4 检索历史库。
8. 若命中历史案例，Step 3/侧栏参考值可在后续状态中替换为数据库样本。

## 渲染边界

MVP 的实时渲染是屏幕模拟，不是物理光谱计算。

允许：

- 根据 Lab 转 CSS 色。
- 用光源预设对 CSS 颜色做轻量偏移，用于表达趋势。
- 用渐变、高光、阴影模拟角度和纹理感。
- 显示不同光源下的简化对比色块。

禁止：

- 声称模拟结果等同实测光谱。
- 声称能从图片或 Lab 得到真实染料配方。
- 新增未验证的复杂光谱算法。

## 错误处理

- Lab 任一轴缺失时，实时渲染区显示「待补全 Lab」。
- 光源未选择时，默认用 D65 预览，但确认参数仍要求人工确认。
- 历史库未命中时，明确 fallback 到 AI 初始提取值。
- AI 图片提示词为空时，显示可编辑空状态，不阻断确认。
- 旧 `confirmedFields` 缺少新增字段时，界面使用默认值，不报错。

## 测试

最小测试范围：

- `buildConfirmationDraft` 能从旧数据和新扩展数据构造草稿。
- `confirmationDraftToRequirement` 能校验新增字段和保留旧字段。
- 历史参考缺失时，参考色块使用 AI 初始 Lab。
- Lab 快捷微调保持数值正确。
- 光源/slider 参数变化能更新确认草稿。
- `npm test` 或最小相关 Vitest 通过。
- `npm run lint` 或项目可用的最小检查通过。

## 不做范围

- 不接入真实测色仪。
- 不新增真实光谱曲线存储。
- 不生成染料生产配方。
- 不接入 ERP/MES。
- 不新增独立历史库管理页面。
- 不把图片提示词做成完整图片上传/识别流程；当前只作为 Step 2/Step 3 的结构化信息展示与编辑。

## 交付标准

- Step 3 字段含义清楚，确认参数不使用暗色背景。
- 图片提示词是紧凑信息条，位于实时渲染下方。
- AI 初始提取值、人工确认实时值、历史库参考值三者视觉上清晰分离。
- 多光源与观察条件可以实时影响预览。
- 保存后的确认 JSON 能支撑后续追踪。
- 旧 demo 数据仍能正常加载。
