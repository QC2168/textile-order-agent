## 一、项目范围现在必须冻结

以四人团队和剩余赛程看，三个原始想法不能同时做成工业级系统。主项目应锁定为“客户色彩需求理解与打样协同”，将硬件扫描、自动配方和 100% 在线质检降级为可展示的软件接口与赛后路线图。

| 层级 | 内容 | 处理原则 |
|---|---|---|
| P0 主闭环 | 客户聊天/截图输入 → AI 结构化 → 缺项追问 → 人工确认 → 历史案例检索 → 打样 Lab/ΔE 对比 → 版本追溯 | 必须完成 |
| P1 加分项 | 参考样布图片/检测文件上传；基布偏暖/偏冷提示；客户确认卡；报告导出 | 有余力再做 |
| P2 路线图 | 分光测色仪、标准光源箱、相机扫描、自动配方、在线全检、ERP/MES 接入 | 只在海报和路演讲 |

> **产品一句话**
>
> 把客户说的“高级一点、别太紫、像上次那块”转成调色师能执行、客户能确认、全过程能追溯的标准色需求与打样任务。

## 四、海报内容可直接套用

| 海报区块 | 建议文案 |
|---|---|
| 标题 | 色译通 ColorBridge——纺织印染客户色彩需求理解与打样协同 Agent |
| 痛点 | 客户语言模糊、关键参数漏传、历史配方难找、打样版本与确认链断裂，最终导致重复打样、返工与客户预期不一致。 |
| 方案 | AI 读取聊天/截图，提取颜色意图与订单字段，主动追问缺项；结合基布信息检索历史案例，录入打样 Lab 后计算色差并形成客户确认与版本追溯。 |
| 演示 | “高级一点的雾霾蓝，别太紫” → 发现 5 个缺项 → 补齐目标 Lab/光源/面料/基布/阈值 → 检索相似案例 → 第一次打样偏黄偏暗 → 第二次达标 → 生成确认卡。 |
| 价值 | 将需求理解前移，减少沟通歧义和重复查找；沉淀老师傅经验；为后续色差仪、扫描设备、自动配方和在线质检预留接口。 |
| 商业化 | 先以轻量 Web 工具进入跟单与打样小组，再按用户/工厂订阅，并提供本地部署、设备接入和 ERP/MES 集成服务。 |

## 五、Demo 必须控制为六次点击

| 点击 | 动作 | 屏幕必须出现 |
|---|---|---|
| 1 | 载入客户聊天 | 显示原始表达：“高级一点的雾霾蓝，别太紫……” |
| 2 | AI 分析 | 出现结构化字段、置信度和缺失项 |
| 3 | 补充确认 | 选择 D65、棉针织、暖底白布、目标 Lab、ΔE 阈值 |
| 4 | 检索历史案例 | 展示 Top 3 及相似原因，不直接声称自动投产配方 |
| 5 | 录入打样结果 | 第一次 ΔE 不合格并提示偏差方向；切换第二次结果达标 |
| 6 | 生成确认/追溯 | 展示客户确认卡、版本链与导出报告 |

> **技术边界口径**
>
> 手机照片和聊天文字只能帮助澄清意图，绝对颜色以标准色卡、Lab 或测色设备数据为准；历史配方是候选依据，最终由调色师确认。这样既专业，也避免评委追问时被“自动配方”反噬。
